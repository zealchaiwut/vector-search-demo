What do these modes, models & scores mean?

Search modes

Semantic
Pure vector (dense) search — matches by meaning using the selected embedding model. Named presets (e.g. dense-only) are available via /api/presets.
Hybrid
Blends dense (semantic) and lexical (keyword) results via RRF fusion — catches both meaning and exact terms.
Keyword
Full-text search (ts_rank) — exact word/stem matching only, no embeddings.

Embedding models

E5 Small (multilingual-e5-small)
384-dim multilingual model; fast and compact, good default for most use cases.
E5 Base / Large
Larger E5 variants with 768 / 1024 dims for higher fidelity at the cost of speed.
BGE-M3
1024-dim model with optional sparse weights; strong cross-lingual performance.
MiniLM (all-MiniLM-L6-v2)
384-dim English-only model; very fast, legacy default.

Explain stages (score pills)

dense
Score from vector (semantic) similarity.
lexical
Score from keyword / full-text match.
rrf
Fused rank score after combining dense + lexical.
rerank
Final cross-encoder relevance score.
